The data layer of the agent internet

Open Primitive
Protocol

A standard for how data providers make their data agent-consumable, verifiable, and discoverable.

The problem

Agents get data naked. No provenance. No freshness guarantee. No confidence score. No license metadata. Every response is a JSON blob with no way to know where it came from, how old it is, or whether to trust it.

The agent internet is missing its data envelope.

The protocol

Three components. Each one solves a piece the agent internet does not have yet.

Manifest
/.well-known/opp.json
Providers declare what data they serve, how fresh it is, and how to verify it. One file. Any domain. Agents discover capabilities before they make a single request.
Response envelope
Every response, every time
Provenance, confidence score, citation chain, and an Ed25519 signature. The data carries its own trust. No external lookup required.
Query endpoint
/v1/query
One standardized interface any agent can call. Structured parameters in, enveloped data out. No bespoke integration per provider.

For agents

Every integration path an agent needs. Pick one or use all of them.

MCP server
13 tools. Drop into Claude, Cursor, or any MCP-compatible client.
A2A agent card
13 skills. Google A2A protocol for agent-to-agent discovery.
llms.txt + llms-full.txt
LLM-readable documentation at the root. Full API spec included.
/v1/ask
Natural language routing. Ask a question, get the right domain and data back.
Citation-ready responses
Every response includes source URL, agency name, and retrieval timestamp.
Reasoning hints
Every response carries context an LLM can use to explain the data.

The vision

Google organized information for humans. Open Primitive organizes data for agents.

The protocol is open. Any data provider can implement it. Federal, state, international, commercial. One envelope. One discovery mechanism. One trust chain.
